# Copy Frame

一个面向页面结构定位场景的 Chrome 扩展。

它可以在网页中高亮当前鼠标所在的 DOM 区域，锁定目标元素后，生成一段适合直接发给 AI 的页面/区域描述文本，方便你表达“要修改或参考页面上的哪一块”。

适合这些场景：

- 告诉 AI 修改某个按钮、输入框、卡片、弹窗或区域
- 快速定位页面中的 DOM 元素并复制稳定描述
- 辅助开发、联调、改版沟通和测试说明

## 功能特性

- 点击浏览器工具栏图标，开启或关闭检查模式
- 鼠标移动时高亮当前可选中的 DOM 元素
- 点击页面元素后锁定选区
- 根据 `data-testid`、`name`、`aria-label`、`id`、类名、文本等信息生成更友好的元素名称
- 自动尝试生成更稳定的定位选择器
- 一键复制可直接发给 AI 的页面/区域描述文本
- 锁定后可通过键盘在同级、父级、子级元素之间快速切换
- 支持撤销到上一次锁定的选区

## 示例

复制后的内容通常类似：

```text
[登录页 / example.com/login] 登录按钮 :: button[data-testid="login-button"]
```

如果目标元素名称过于泛化，插件会自动补充父级上下文，帮助 AI 更准确理解位置。

## 本地开发

在项目根目录执行：

```bash
npm install
npm run typecheck
npm run build
```

构建完成后会生成可加载的 `dist` 目录。

## 加载到 Chrome

1. 打开 `chrome://extensions/`
2. 打开右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择项目下的 `dist` 目录
5. 加载成功后，就可以在工具栏看到 Copy Frame 图标

## 使用流程

1. 点击 Copy Frame 图标，进入检查模式
2. 将鼠标移动到目标区域，查看高亮框和名称提示
3. 点击页面中的目标元素，锁定选区
4. 点击高亮框右上角的 `Copy` 按钮，或使用快捷键复制描述
5. 将复制结果直接粘贴到 AI 对话中，再补充你的修改目标或参考说明

## 快捷键

| 快捷键 | 作用 |
| --- | --- |
| `Alt + C` | 启动或关闭插件检查模式 |
| `Ctrl + C` / `Cmd + C` | 复制当前锁定区域描述 |
| `Ctrl + Z` / `Cmd + Z` | 撤销到上一次锁定结果 |
| `Tab` | 切换到下一个同级可选元素 |
| `Shift + Tab` | 切换到上一个同级可选元素 |
| `Enter` | 进入当前元素的更细一层 |
| `Shift + Enter` | 切换到当前元素的上一层 |
| `Esc` | 退出当前锁定 |
| 再按一次 `Esc` | 关闭插件检查模式 |

## 使用建议

- 优先选择带有 `data-testid`、`name`、`aria-label` 或 `id` 的元素
- 如果选区太大，可以按 `Enter` 进入更细一层
- 如果选区太小，可以按 `Shift + Enter` 回到父级
- 如果同级有多个相似元素，可以用 `Tab` / `Shift + Tab` 快速切换
- 复制后建议补充一句目标说明，例如“请把这里改成圆角按钮”

## 当前限制

- 目前主要面向 DOM 元素定位，不适合识别纯画布内容
- `canvas` 区域暂不支持稳定命名
- 一些浏览器受限页面或特殊跨域页面，可能无法正常注入或工作
- 生成的描述以“稳定定位 + 友好命名”为主，不等同于完整视觉截图理解
