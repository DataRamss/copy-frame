# Copy Frame 插件使用说明

## 1. 插件简介

Copy Frame 是一个面向页面结构定位场景的 Chrome 插件。
它可以在网页中高亮当前鼠标所在的 DOM 区域，锁定目标元素后，生成一段适合直接发给 AI 的页面/区域描述文本，便于你表达“要修改或参考哪一块页面”。

适合的场景包括：

- 向 AI 描述网页上某个按钮、输入框、卡片、弹窗或区域
- 快速定位页面中的 DOM 元素并复制稳定描述
- 辅助开发、联调、页面改版沟通

---

## 2. 主要能力

插件当前支持：

- 点击 Chrome 工具栏中的插件图标，开启或关闭检查模式
- 鼠标移动时高亮当前可选中的 DOM 元素
- 点击页面中的元素后锁定选区
- 根据元素的 `data-testid`、`name`、`aria-label`、`id`、类名、文本等信息生成友好的元素名称
- 自动尝试生成更稳定的定位选择器
- 一键复制可直接发给 AI 的页面/区域描述文本
- 在锁定后通过键盘在同级、父级、子级元素之间快速切换
- 支持撤销到上一次锁定的选区

---

## 3. 安装与加载

如果你已经有打包后的 `dist` 目录，可以直接加载；如果没有，可以先构建项目。

### 3.1 本地构建

在项目根目录执行：

```bash
npm install
npm run typecheck
npm run build
```

构建完成后，会生成可加载的 `dist` 目录。

### 3.2 加载到 Chrome

1. 打开 Chrome 扩展管理页：`chrome://extensions/`
2. 打开右上角的“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择项目下的 `dist` 目录
5. 加载成功后，你会在工具栏看到 Copy Frame 图标

---

## 4. 基本使用流程

### 4.1 开启检查模式

点击 Chrome 工具栏中的 Copy Frame 图标后，插件进入检查模式：

- 页面上会出现粉色高亮框
- 鼠标移动到哪里，就会高亮当前命中的可选 DOM 区域
- 标签上会显示插件推断出的元素名称

### 4.2 锁定目标区域

将鼠标移动到目标区域后，直接点击：

- 当前区域会被锁定
- 右上角会出现 `Copy` 按钮
- 插件进入“已锁定”状态

### 4.3 复制描述文本

锁定后，你可以：

- 点击高亮框右上角的 `Copy` 按钮
- 或使用复制快捷键（见下文）

复制后的内容通常类似：

```text
[登录页 / example.com/login] 登录按钮 :: button[data-testid="login-button"]
```

如果目标元素本身名字比较泛，插件会自动补充父级上下文，帮助 AI 更准确理解目标位置。

## 5. 快捷键说明

> `Alt + C` 是全局开关；下面其余快捷键只有在插件已经开启，并且页面处于检查/锁定状态时才生效。

| 快捷键 | 作用 | 说明 |
| --- | --- | --- |
| `Alt + C` | 启动或关闭插件检查模式 | 检查中或已锁定时也可直接关闭；输入框/富文本中默认忽略 |
| `Ctrl + C` / `Cmd + C` | 复制当前锁定区域描述 | 需要先锁定一个元素 |
| `Ctrl + Z` / `Cmd + Z` | 撤销到上一次锁定结果 | 只能在已锁定状态下使用 |
| `Tab` | 切换到下一个同级可选元素 | 在兄弟节点之间前进 |
| `Shift + Tab` | 切换到上一个同级可选元素 | 在兄弟节点之间后退 |
| `Enter` | 进入当前元素的更细一层 | 选择当前元素的可选子元素 |
| `Shift + Enter` | 切换到当前元素的上一层 | 选择当前元素的可选父元素 |
| `Esc` | 退出当前锁定 | 从“已锁定”回到“检查中” |
| 再按一次 `Esc` | 关闭插件检查模式 | 完全退出插件高亮状态 |

### 5.1 关于浏览器级快捷键
当前版本临时内置了 `Alt + C` 作为页面级开关快捷键。
默认操作方式是：

- 按 `Alt + C` 开启或关闭插件
- 或点击 Chrome 工具栏中的 Copy Frame 图标来开启或关闭插件

---

## 6. 使用建议

为了让复制出来的结果更稳定、更适合发给 AI，建议：

- 尽量优先选择带有 `data-testid`、`name`、`aria-label` 或 `id` 的元素
- 如果第一次点中的区域太大，可以按 `Enter` 进入更细一层
- 如果点中的区域太小或太具体，可以按 `Shift + Enter` 回到父级
- 如果同级有多个相似元素，可以用 `Tab` / `Shift + Tab` 快速切换
- 复制后可直接粘贴到 AI 对话中，并补充你的修改目标或参考说明，例如“请把这里改成圆角按钮”

---

## 7. 当前限制

当前版本的限制包括：

- 主要面向 DOM 元素定位，不适合识别纯画布内容
- `canvas` 区域目前不支持复制稳定命名
- 一些浏览器受限页面、特殊跨域页面，可能无法正常注入或工作
- 生成的描述以“稳定定位 + 友好命名”为主，不等同于完整的视觉截图理解

---

## 8. 推荐使用场景

推荐在这些场景使用：

- 页面改版时，告诉 AI “修改这一块”
- 联调时，快速确认某个交互区域对应的 DOM 节点
- 让产品、测试、开发更容易表达页面目标区域
- 为自动化、测试、AI 辅助开发提供更明确的定位描述

---

## 9. 一句话总结

Copy Frame 的核心用法可以概括为：

**点击插件图标开启 -> 鼠标悬停查看 -> 点击锁定 -> 按 `Copy` 或 `Ctrl/Cmd + C` 复制描述。**
